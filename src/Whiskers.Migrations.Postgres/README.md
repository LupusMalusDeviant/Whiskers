# Whiskers.Migrations.Postgres

The **PostgreSQL** EF Core migrations for `MetricsDbContext` (which lives in [`Whiskers.Data`](../Whiskers.Data/)) — a parallel assembly to [`Whiskers.Migrations.Sqlite`](../Whiskers.Migrations.Sqlite/) with its own model snapshot ([ADR-0004](../../docs/adr/0004-postgres-provider-support.md)).

- `Migrations/<timestamp>_InitialCreate` (+ `.Designer`) — the provider-native PG schema: `bigint GENERATED BY DEFAULT AS IDENTITY` primary keys, `timestamp with time zone` for every `DateTime` (via the UTC converter), plus `text`, `boolean`, `double precision`. Independent of the SQLite baseline, so it carries its own timestamp id.
- `Migrations/MetricsDbContextModelSnapshot` — the PG model snapshot.

The app pins this assembly with `UseNpgsql(..., o => o.MigrationsAssembly("Whiskers.Migrations.Postgres"))`.

Scaffold a new migration — the design-time factory branches on `WHISKERS_DB_PROVIDER`:
```
$env:WHISKERS_DB_PROVIDER="postgres"
dotnet ef migrations add <Name> --project src/Whiskers.Migrations.Postgres --startup-project src/Whiskers
```

To apply against a real PostgreSQL, the overlay [`deploy/docker-compose.postgres.yml`](../../deploy/docker-compose.postgres.yml) starts a server and points Whiskers at it. References `Whiskers.Data` only — never the app.
