# Product

Product-strategy documents for Whiskers: the canonical positioning, message hierarchy, and
(over time) demo and screenshot references. Engineering, docs, and the public website should draw
their product language from here so README, website and in-app help stay consistent.

| Document | What |
|---|---|
| [POSITIONING.md](POSITIONING.md) | Canonical one-liner, elevator pitch, target audience, problem, value proposition, the three product pillars, the hero workflow, approved/avoided wording, and the glossary. |
| [demo-script.md](demo-script.md) | The hero-workflow demo (Observe → Propose → Check policy → Approve → Execute → Verify → Audit) and how to build the anonymized demo dataset. |
| [screenshots.md](screenshots.md) | Canonical product screenshots mapped to the three pillars, capture conventions, and anonymization rules. Images live in [screenshots/](screenshots/). |

> These are the source of truth for *how we talk about Whiskers*, not a feature backlog. Product
> and roadmap work items live under [../roadmap/](../roadmap/); architecture and security live in
> [../ARCHITECTURE.md](../ARCHITECTURE.md) and [../../SECURITY.md](../../SECURITY.md).
